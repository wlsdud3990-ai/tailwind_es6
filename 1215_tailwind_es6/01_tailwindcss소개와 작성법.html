1. Tailwind CSS소개와 설치

css역사 - html언어탄생(1990) > css스타일 언어 탄생(1996) > css Zen Graden의 열풍(2003 selector의 복잡성) > sass의 등장(2006), 전처리기 > Bootstrap(프레임워크) > Tailwind CSS(프레임워크)

Tailwind CSS는 'Utility-First 컨셉'을 가진 CSS 프레임워크로서 부트스트랩과 비슷하게 m-1, flex와 같이 미리 세팅된 '유틸리티 클래스'를 활용하는 방식으로 HTML 코드 내에서 스타일링을 할 수 있습니다.

HTML태그 요소에 직접 클래스 이름을 추가하여 스타일을 적용하는 방식이며, CSS파일 작성을 최소화하고 개발 속도를 향상시킬 수 있습니다.

pt(padding-top), mb(margin-bottom), text-sky(500)

2. Tailwind CSS의 장점

  • Utility-First의 편리함과 빠른 개발 - Utility-First 컨셉은 Tailwind CSS의 메인 컨셉이자 가장 큰 장점이다. 매우 쉽고 빠르게 원하는 디자인을 개발할 수 있다.
  • css 코드도 HTML 코드 안에 있기 때문에 HTML와 CSS 파일을 별도로 관리할 필요가 없다. 그래서 HTML, CSS를 왔다 갔다하며 시간을 쓰거나 화면을 분할해서 사용하지 않아도 된다.
  • 로우 레벨의 스타일 제공 - 각 CSS 요소 수준의 '유틸리티 클래스'를 제공하기 때문에 세밀하게 원하는 디자인을 구현할 수 있다. (margin-top:10px; => mt-10)
  • JavaScript 코드와의 분리 - JavaScript 코드와 완전히 분리되어 있다. 그러므로 프로젝트 진행 도중 JavaScript 프레임워크를 변경하여도 큰 추가 작업 없이 기존의 HTML 코드를 그대로 쓸 수 있다.
  • 클래스명 이름짓기? BEM, OOCSS 등의 방법론이 나올 정도로 클래스명 짓는 일은 까다롭다. Tailwind CSS를 사용하면 랩핑 태그의 클래스명을 사용할 일이 거의 없으므로 container, wrapper, inner-wrapper와 같은 클래스명을 고민하지 않아도 된다.

3. Tailwind CSS의 단점

  • 부트스트랩처럼 태그 안의 코드가 점점 많아짐. 지저분한 코드
  • 클래스명 숙지해야 - 초반에는 각 스타일의 클래스명을 익히느라 개발하는 내내 문서를 참고해야 하는 번거로움이 있다. 그래도 대부분의 클래스명이 기존 CSS 속성이나 속성값과 비슷한 경우가 많고 자동완성을 지원하는 'Tailsind Intelli Sense 플러그인'이 있어서 금방 익숙해지기는 한다.
  • 자바스크립트 코드 사용불가 - 클래스명을 분기 처리하여 '동적으로 스타일링'을 설정할 수는 있지만 styled-components와 같이 JavaScript 변수 값에 따라 가로 길이를 설정하는 등의 구현은 가능하기는 하지만 무척 번거로운 설정이 필요하다.
  • HTML와 CSS코드 혼합되어 복잡한 코드를 보여준다.

4. Tailwind CSS 설치

  • 사용하는 프레임워크나 도구에 따라 설치 방법이 조금씩 다르다. 설치는 'npm, yarn' 등의 패키지 매니저나 CDN을 통해 가능합니다.(리액트 공식 문서 참조)
  • 방법1 : header영역에 script src="https://cdn.tailwindcss.com" 주소 추가하여 사용
  • 방법2 : npx create-react-app [프로젝트 이름] 명령어를 통해 리액트 앱을 생성한 다음 'npm install -D tailwindcss', 'npx tailwindcss init' 명령어로 실행

5. Tailwind IntelliSense 플러그인 설치

  • VS Code Extention인 IntelliSense를 설치하면 훨씬 빠르고 쉽게 스타일을 적용 가능하다.
  • 미리보기, 신텍스 하이라이팅, 링팅, 자동완성, 오류, 버그를 강조해주고 클래스 이름에 마우스를 hover시 해당 클래스가 가지는 스타일을 보여주는 편리한 Extention이니 Tailwind를 쓴다면 같이 사용해 주는 게 좋다.
